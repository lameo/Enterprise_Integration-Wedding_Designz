Scenario
1.
User completes orders at Wedding Designz website with the following fields filled:
 Personal Information
 Venue
I. User selects a country and click the “Find Venue” button, which IM receives via
HTTP, triggering the Get Location Process:
i.
Content-based routing occurs here where the Venue Manager that receives the
JMS Message is dependent on the country selected by the user. (either
SGprocess, AUprocess or ITprocess).
ii. At the same time, a query is sent to Google API to retrieve the top wedding
locations of the selected country.
iii. The IM combines the information from the Venue Manager with that from Google
API into a JSON object which is sent back to the webpage as a HTTP response
iv. The webpage displays the list of available locations and allow the customer to
choose one or multiple wedding locations.
 Boutique
II. User can pick a boutique location and date for their dress-fitting
IM receives the data via HTTP, triggering the Get Availability Process:
i.
IM triggers sends a HTTP request to the Boutique REST API (written in Python)
and returns a JSON object containing the available time slots back to the web
page.
ii. The webpage displays a list of available time slots and allow the customer to pick
one.
2.
When the user clicks submit, a HTTP request is sent to the IM, triggering the “Book
Process” where an unconfirmed booking entry is created in the database of the Boutique
Manager’s database. At the same time, the user is redirected to the confirm booking page.
3.
User clicks the Paypal checkout button to pay a down payment to confirm their booking,
triggering the “Create Payment” process:
A. IM invokes the Paypal API to get a token, then uses it to create the payment with the
relevant details.
B. The web page displays Paypal login page where the user has to login with their Paypal
account and approve the payment.
4.
When user clicks pay now, the payment is approved and executed. The user is then
redirected to the successful booking page.

After the wedding is completed
1. The photographer uploads the photos as a zip file onto the company’s FTP server.
2. An email is sent to the customer containing a link for them to download their wedding
photos.
3. After the customer clicks the link, he/she is directed to a page where there is a button to
download their images.
A. IM parses the customer details from JSON format to XML format and simultaneously
proceed to 3 processes (Facebook, twitter and FTP transfer)
B. At the same time, a congratulatory message for the couple is posted on the company’s
Facebook and Twitter pages.
*All the processes have error handling, if IM catch an error, it will write file, render
XML(error) into JSON format and send back to the webpage.

Scenario Walkthrough
1. User completes orders at Wedding Designz website with the following fields filled:
A. Personal Information
B. Venue
1) User selects a country and click the “find venue” button, which IM receives via HTTP,
triggering the Get Location Process:
i.
Content-based routing occurs here when IM queries from EIproject database for
the country name and its corresponding queue name. A JMS Request Message is
sent to the queue name, triggering one of the venue processes (either SGprocess,
AUprocess or ITprocess).
ii.
In the respective venue process triggered, IM reads the specified locations CSV
file, parse its data, and remove unavailable places.
1. IM renders the locations as XML and send the reply as a JMS message back
to GetLocationProcess.
iii. Concurrently, IM queries from the EIproject database to retrieve the Google API
Key and invoke the Google Place Search API. IM parses the results into XML
format.
iv. IM parses the XML data received from the venue management system. Thereafter,
IM renders the results from venue process and Google API from XML format into
JSON. Data transformation occurs here when rendering JSON.
7 | P a g e
v. IM sends the JSON data back to the webpage as a HTTP response.
vi. The webpage displays the list of available locations and allow the customer to
choose one or multiple wedding locations.
Blue: From Venue Management System
White: From Google Place Search API
C. Boutique
1) User can pick a boutique location and date for their dress-fitting
i.
IM receives the data via HTTP, triggering the “Get Availability Process”:
1. IM triggers the Call Process in the Get Availability Process, triggering the
Use Boutique API Process, invoking Boutique REST API (written in Python)
and returns the available time slot in JSON format. IM then uses java code to
format the output and send it to the webpage as HTTP response.
2. The webpage displays a list of available time slots and allow the customer to
pick one.
2) When user clicks submit, IM receives the order details via HTTP, triggering the “Book
Process”:
i.
IM triggers the Call Process. Call Process triggers the Use Boutique API Process,
invoking Boutique REST API. Our boutique system will save the order in its
BookingOrder database and return the Booking ID in JSON format. IM then uses
java code to format the output and send it to the webpage as HTTP response.
3) The webpage displays a confirm booking page for the customer to make payment.
8 | P a g e
4) User clicks the Paypal checkout button, triggering the “Create Payment” process:
i.
IM invokes the Paypal Access Token API to get a token. It then parses the token
from JSON format into XML format and read paypal.json file.
ii.
IM invokes Paypal Create Payment API to create a payment ID. IM renders it as
JSON format and send it back to webpage as HTTP response.
iii. The web page displays Paypal login page.
5) From the Paypal login page that allows users to log in with their credentials to the
“pay now” button, is the Approve Process that is done by Paypal.
6) Execute Payment process which includes confirmation of boutique booking.
9 | P a g e
i.
IM renders payment ID in JSON format and invokes Paypal Execute Payment API
to get the payment information.
ii.
IM triggers the Confirm Booking process, triggering our Use boutique API process.
This invokes our boutique REST API to update the booking status. IM will send a
confirmation email to the customer and a confirm booking successful message to
the webpage via HTTP response.
10 | P a g e
After the wedding is completed
1. The photographer uploads the photos as a Zip file onto the company’s FTP server.
2. The photographer manually sends an email to the customer a link for them to download
their wedding photos.
3. After the customer clicks the link, IM receives the customer’s booking ID via HTTP,
triggering the download photo and publish to social media process:
A. IM triggers the “Get User Detail” process in download photo and publish to social media
process, invoking Boutique REST API. The boutique system queries from
“bookingorder” database to retrieve the couple names, venue and photos filename of
the booking ID, sending them back to download photo and publish to social media
process.
B. IM parses the customer details from JSON format to XML format and simultaneously
proceed to 3 processes (Facebook, twitter and FTP transfer)
I. Facebook
i.
IM retrieves the company’s Facebook access token from EIproject database.
ii. IM invokes Facebook’s Graph API, to retrieve Facebook ID.
iii. IM parses the JSON response into XML format.
iv. A unique congratulatory message is posted to the couple on our Facebook
profile.
II. Twitter
i.
IM authenticates our twitter account using ‘OAUth 1.0 Resource’.
ii. After authentication, a congratulatory message for the couple is posted on our
Twitter account.
11 | P a g e
III. FTP transfer
i.
IM uses the file-name to locate the file on the photographer local drive.
ii. IM transfers the file onto the company’s FTP server. The sleep activity is used
here as the transfer process takes time.
iii. The Send File activity is started, transferring the file towards the Webpage
resources folder.
iv. IM sends a link via HTTP response. The link will redirect the user to download the
file.
v. Webpage shows a button/link that lets user download their photos.
*All the processes have error handling, if IM catch an error, it will write file, render
XML(error) into JSON format and send back to the webpage
